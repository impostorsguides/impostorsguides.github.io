While making a simple to-do app in Django as part of a guide that ClaudeAI wrote for me, I came across the following code:

...

class TodoListView(ListView):
  model = Todo
  template_name = 'todo_app/todo_list.html'
  context_object_name = 'todos'

...

Coming from Ruby, this code was strange-looking to me. It looked as if we're declaring a class, and then declaring several local variables inside the class. That's not something you'd typically do in Ruby. Instead, you could declare class variables.

I assumed that's what was happening here, but I wanted to be sure.
